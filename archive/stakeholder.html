The Mechanism

How Baskets will work.

01
Manager Publishes Update

A portfolio manager publishes a change to their Substack model portfolio — adding a new position, adjusting weights, or removing a holding. Baskets detects the update.

02
Weights Recalculated

The Baskets strategy engine parses the update, diffs it against the current model, and generates new target percentage allocations. The updated weights are pushed to IBKR via the TWS API.

03
Accounts Auto-Rebalance

IBKR's model portfolio system receives the new weights and automatically generates rebalance orders for all subscribed client accounts — executing trades across 170+ global exchanges.

Roadmap

Development phases.

A disciplined, phased approach from algorithm validation to market launch.

Complete
Phase 1: Algorithm & Analysis

Minimum capital vs. tracking error optimisation. Capital efficiency modelling across the Citrindex portfolio. Binding constraint analysis (Japanese lot sizes on TSE requiring 100-share minimums).

Current
Phase 2: Architecture & Partnerships

Finalise RIA partner selection. Establish sub-advisory agreement. Set up IBKR model portfolio (static weights). Test TWS API integration for weight updates. Build prototype strategy engine (weight calculator).

Next
Phase 3: Product Build

User-facing web app (basket selection, performance dashboard). IBKR TWS API integration for automated weight pushing. Portfolio manager update detection system. User onboarding flow via RIA partner.

Planned
Phase 4: Launch

Soft launch with first basket (Citrindex Mirror). Invite-only "Batch 1" for early adopters. Iterate on UX based on user feedback.

Future
Phase 5: Scale & Expand

Add multiple baskets (Le Shrub, Eliant, etc.). Baskets Co-Pilot (AI engine) development. Monitor Alpaca global stocks launch for potential infrastructure migration. Evaluate on-chain tokenisation layer for global access. Consider Reg A+ offering or ETF wrapper at significant scale.
